# AI Agent 产品方案文档

> **产品定位**：一个"帮你看视频"的 AI 助手
>
> **一句话描述**：丢给它一个抖音/YouTube链接，它替你看完，然后你可以问任何问题。
>
> **创建日期**：2026-07-11
>
> **当前阶段**：Demo MVP 已完成，Phase 2 正式上线规划中

---

## 一、产品定位与核心价值

### 定位
不是泛化的"多模态知识助手"，而是专注解决一个高频刚需——**"视频太长/太水，我只想知道讲了什么"**。

### 为什么这个切入点是对的
- 抖音和YouTube覆盖了几乎所有年龄段的视频消费者
- "不想看/没时间看但需要知道内容"是普适需求，不分年龄、职业
- 翻译功能天然跨语言，中文用户想看英文YouTube视频，翻译就是刚需入口
- 视频内容质量参差不齐，用户越来越需要AI帮ta"预筛选"

### 产品灵魂
> **不是让用户更会搜索，而是让用户更会提问。**

## 二、MVP 功能定义

### 核心功能（仅此三个）

| 功能 | 输入 | 输出 | 体验目标 |
|------|------|------|----------|
| 视频总结 | 抖音/YouTube链接 | 结构化摘要（分段标题 + 每段要点）+ 总时长 | 用户10秒看完摘要，判断要不要深入 |
| 关键信息提取 | 视频链接 + 关注点 | 结构化列表，每项带时间戳 | 用户像搜索一样精准定位视频内容 |
| 跨语言翻译 | 外语视频链接 | 中文字幕/中文总结 | 看外语视频像看母语视频一样自然 |

### 明确不做（防止范围蔓延）
- ❌ 不做图片上传
- ❌ 不做纯文字粘贴（后期再加）
- ❌ 不做跨视频关联
- ❌ 不做实时对话（第一版只做单轮问答+简单追问）
- ❌ 不做移动端App（先用Web验证）

## 三、目标用户画像

| 场景 | 典型用户 | 典型行为 | 核心需求 |
|------|---------|---------|---------|
| 学习场景 | 学生/自学者 | 看教程视频前先看总结 | "这个30分钟的教程到底讲没讲我想学的那部分？" |
| 信息获取 | 上班族/刷视频的人 | 刷到长视频但没时间看 | "这个15分钟的视频能不能3句话说完？" |
| 跨语言消费 | 所有人 | 看到外语视频想了解内容 | "这个英文/日文/韩文视频在说什么？" |
| 内容创作参考 | 自媒体/创作者 | 快速了解竞品/同行的视频内容 | "最近爆火的这类视频都在讲什么？" |

**MVP阶段优先服务前两个场景**（学习+信息获取），翻译功能作为拉新抓手。

### 5.2 视频处理流水线

```
Step 1: 链接解析 (1-3秒)
    ↓
Step 2: 音频下载 + ASR转写 (视频时长的20%-30%)
    ↓
Step 3: 智能关键帧抽取 (视频时长的10%-15%)
    ↓
Step 4: AI 理解与生成 (10-60秒)
```

### 5.3 技术选型

| 组件 | 方案 | 备注 |
|------|------|------|
| 视频下载/解析 | yt-dlp | 成熟稳定，支持2000+网站 |
| ASR引擎 | 硅基流动 SenseVoice（云端）/ faster-whisper（本地） | 双模切换，云端优先 |
| 关键帧抽取 | FFmpeg + 场景检测 | 通用且高效 |
| AI 大模型 | DeepSeek API（默认）/ Claude API（可选） | LLM_PROVIDER 环境变量切换 |
| 翻译 | LLM 一步完成 | 减少流水线环节 |
| 任务队列 | 暂无（MVP 同步处理） | Week 5-6 引入 BullMQ + Redis |
| 存储 | 临时文件系统 | 处理后自动清理 |
| 前端框架 | Next.js 16 + React 19 + Tailwind CSS v4 | SSR友好，生态成熟 |

## 六、成本估算

### 单个 10 分钟视频
| 环节 | 成本 |
|------|------|
| 音频下载带宽 | 约15MB（可忽略） |
| ASR (Whisper) | 约$0.01（本地GPU摊销） |
| 关键帧抽取 | 10-20张图片（可忽略） |
| Claude API 输入 | 约12,000 tokens |
| Claude API 输出 | 约500-1,500 tokens |
| **单视频总成本** | **约 $0.20-0.30** |

## 七、付费模型

### 免费版
- 每日额度：3个视频
- 单视频时长限制：≤30分钟
- 支持平台：YouTube、抖音
- 功能限制：仅支持总结和翻译，不支持自定义提问
- 免费用户高峰期排队
- 有水印/品牌露出

### 专业版（$9.9/月 或 ¥29/月）
- 每日额度：50个视频
- 单视频时长：≤3小时
- 支持平台：全部
- 所有功能：总结、关键提取、翻译、自定义提问、多轮追问
- 优先处理：专属计算资源
- 无水印
- 导出功能（Markdown/PDF）
- 历史记录永久保存

### 定价策略
- 年付折扣：¥199/年（约¥16.6/月）
- 价值锚定"时间价值"：省了看视频的时间
- 核心转化逻辑：免费用户→体验价值→频次不够用→付费

## 九、Prompt 设计原则

### 总结类
- 要求模型"先列出视频结构和分段"，再总结每段
- 要求"如果视频是教程/评测/新闻类，请按对应的最佳结构输出"
- 限制输出长度

### 关键信息提取
- 先判断视频类型，再提取对应维度的关键信息
- 教程类→知识点、步骤、工具
- 新闻类→5W1H
- 评测类→优缺点、对比对象、结论

### 翻译
- 不是字面翻译，是"理解后用目标语言重新表达"
- 保留原文语气和风格

#### 2.1 Vercel 平台部署

**部署方案：**

| 组件 | 方案 | 说明 |

**待解决问题：**

#### 2.2 Bilibili 视频链接支持

**目标：** 新增 B 站视频链接解析，覆盖中文视频内容场景。

**技术要点：**

| 事项 | 说明 |
|------|------|
| 解析方式 | 使用 yt-dlp 的 Bilibili 提取器 |
| 必要条件 | Netscape 格式 cookies（`BILIBILI_COOKIES` 环境变量），包含 `SESSDATA` 和 `buvid3` |
| User-Agent | 需伪装 Chrome 浏览器 UA |
| 海外访问 | B 站可能屏蔽海外 IP，需测试（Vercel 服务器在海外） |
| 降级方案 | 如 yt-dlp 被 412 拦截，可尝试 Playwright 方案 |

**开发任务：**
- [ ] `src/lib/url-utils.ts`：添加 Bilibili URL 正则匹配
- [ ] `src/lib/video-processor.ts`：B 站链接走 yt-dlp 下载 + cookie 注入
- [ ] `src/app/api/process/route.ts`：添加 B 站分支处理逻辑
- [ ] 测试：国内 IP 直连、海外 IP + cookie 两种场景
- [ ] 前端 UI：更新平台提示文字

**里程碑：** 支持 YouTube + 抖音 + Bilibili 三大平台。

#### 2.3 会员系统

**目标：** 引入用户注册/登录和额度管理，实现免费 + 付费双模式。

**会员体系设计：**

| 维度 | 免费用户 | 专业会员（¥29/月） |
|------|---------|-------------------|
| 每日视频额度 | 3 个/天 | 50 个/天 |
| 单视频时长限制 | ≤30 分钟 | ≤3 小时 |
| 支持平台 | YouTube、抖音、B站 | 全部 |
| 功能范围 | 视频总结 + 追问 | 全部功能 |
| 处理优先级 | 高峰期排队 | 优先处理 |
| 历史记录 | 保留 7 天 | 永久保存 |
| 导出功能 | 无 | Markdown / PDF |
| 品牌露出 | 有水印 | 无水印 |

**技术实现：**

| 模块 | 方案 | 说明 |
|------|------|------|
| 用户系统 | NextAuth.js + GitHub OAuth | 最简 MVP 方案，后续可加邮箱/微信登录 |
| 数据库 | Vercel Postgres / Supabase | Serverless 友好，免费额度足够 MVP |
| 额度管理 | 每日重置计数器 | 数据库记录 `daily_count` + `last_reset_date` |
| 付费 | Stripe（海外）/ 支付宝（国内） | Phase 1 先做 Stripe，后续接入支付宝 |
| 会员状态 | 数据库 `subscription` 表 | 记录 `plan`、`expires_at` 等字段 |

**数据库表设计（草案）：**

```sql
-- 用户表
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  email VARCHAR(255) UNIQUE,
  name VARCHAR(100),
  avatar_url TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 额度表
CREATE TABLE quotas (
  user_id INTEGER PRIMARY KEY REFERENCES users(id),
  daily_count INTEGER DEFAULT 0,
  last_reset_date DATE DEFAULT CURRENT_DATE,
  total_processed INTEGER DEFAULT 0
);

-- 会员表
CREATE TABLE subscriptions (
  user_id INTEGER PRIMARY KEY REFERENCES users(id),
  plan VARCHAR(20) DEFAULT 'free',  -- free | pro
  stripe_customer_id VARCHAR(100),
  stripe_subscription_id VARCHAR(100),
  expires_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW()
);
```

## 十二、已知风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 抖音反爬升级 | 核心功能不可用 | 备用方案（第三方服务、模拟浏览器、官方API） |
| 视频下载速度慢 | 用户流失 | 预下载缓存 + 流式处理 |
| ASR对网络用语识别差 | 中文理解质量下降 | ASR后加LLM纠错层 |
| 用户期望管理 | 差评 | 产品文案诚实，明确AI主要依赖字幕 |
| 版权灰色地带 | 法律风险 | 处理后立即删除原始文件，ToS明确用户责任 |
| 幻觉问题 | 编造视频不存在的内容 | 每个结论绑定时间戳/原文引用，可验证性设计 |
