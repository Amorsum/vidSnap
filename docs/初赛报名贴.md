## 1. Demo 简介

**VidSnap** 是一个 AI 视频总结网站，粘贴 YouTube 或抖音链接，AI 自动替你看完视频，生成中文摘要，还能基于视频内容追问。

**核心用户：** 想快速获取视频核心信息的人——学生看网课、上班族追技术分享、普通用户刷长视频，都不用从头看到尾。

**主要功能：**

1. **一键视频总结** — 粘贴 YouTube/抖音链接，自动下载音频 → 语音转写 → AI 总结，输出结构化摘要和分段要点
2. **智能追问对话** — 基于视频原文内容，可以问任何问题（"这个观点在视频哪里提到的？"），AI 会结合原文回答
3. **跨语言理解** — 日语、韩语、英语视频，自动输出中文总结

## 2. Demo 创作思路

**灵感来源：** 每天刷到很多「干货视频」，但大部分 15 分钟的视频，真正有用的就 2-3 分钟。一个个点开看太费时间，不看又怕错过。我就想：能不能让 AI 替我看？

**想解决的问题：** 视频内容消费效率低。用户需要快速判断一个视频值不值得看，或者只想看核心要点，而不是从头刷到尾。

**为什么做这个方向：** 市面上有文本总结、图片总结，但视频总结还不多。而且视频是最丰富的信息载体（有语音、有画面、有文字），AI 刚好能在这个场景发挥最大价值。先做 YouTube 和抖音，是因为这两个平台内容量最大、用户需求最刚。

## 4. TRAE 实践过程

### 完整开发流程

整个项目从零到 Demo MVP 完成，全部使用 TRAE 进行开发，主要经历了以下阶段：

**阶段一：项目搭建与环境配置**
- TRAE 帮我搭建了 Next.js 16 项目结构，配置了 TypeScript + Tailwind CSS
- 验证了 yt-dlp、FFmpeg、Whisper 等本地依赖
- 配置了 LLM 服务层（支持 DeepSeek/Claude 双模型切换）

**阶段二：核心流水线开发**
- 实现 YouTube 链接 → 音频下载 → Whisper 转写 → AI 总结 的完整链路
- 遇到 DeepSeek 输出日语/韩语的问题，TRAE 帮我优化了 Prompt 约束
- 添加了 SSE 流式响应，实现实时进度展示

**阶段三：多平台支持**
- 添加抖音视频支持，TRAE 帮我用 Playwright 绕过了抖音的 X-Bogus 签名
- 实现了 URL 自动提取（从混杂的分享文案中识别链接）

**阶段四：部署与优化**
- 尝试了 Vercel、Netlify 部署，遇到二进制依赖和 bot 检测问题
- 最终用 Cloudflare Tunnel 解决了公网访问问题
- 优化了转写精度（切换到 SenseVoice 云端 API）
